## Executive Summary

GitHub Actions-based AI orchestration system that transforms GitHub events (issues, PR comments, reviews) into autonomous work orders. Uses devcontainers + opencode CLI to run AI agents. This is a **template repository** - new projects are created via `nam20485/workflow-launch2` automation.

## CI/CD Pipeline

### validate.yml (on push/PR)
- **lint**: actionlint, hadolint, shellcheck, PSScriptAnalyzer, JSON validation
- **scan**: gitleaks secret scanning
- **test-prompt-assembly**: Prompt assembly tests
- **test-image-tag-logic**: Image tag resolution tests
- **test-pester**: PowerShell tests (dry-run)
- **test-devcontainer-build**: Build devcontainer + smoke tests

### publish-docker.yml (on push to main)
- Builds `.github/.devcontainer/Dockerfile`
- Pushes to `ghcr.io/${{ github.repository }}:main-latest`
- Signs with cosign/sigstore

### prebuild-devcontainer.yml (on workflow_run)
- Layers devcontainer Features on published image
- Pushes to `ghcr.io/${{ github.repository }}/devcontainer:main-latest`

### orchestrator-agent.yml (on GitHub events)
- Assembles prompt from event data
- Pulls prebuilt devcontainer image
- Runs `opencode --agent Orchestrator` in devcontainer

## Code Style & Conventions

### Python (ruff + mypy)
- Target: Python 3.12+
- Line length: 100 characters
- Strict mypy enabled
- Rules: pycodestyle (E/W), pyflakes (F), isort (I), bugbear (B), comprehensions (C4), pyupgrade (UP)

### Shell Scripts
- Must pass `shellcheck --severity=warning`
- Use `set -euo pipefail`

### Workflows
- Pin actions by SHA (not tag)
- Single `name:`, `on:`, `jobs:` per file

### General
- Keep changes minimal and targeted
- Never hardcode secrets/tokens
- Preserve `{{__EVENT_DATA__}}` placeholder in prompt template

## Common Pitfalls

1. **Missing devcontainer image**: On fresh clones, run `publish-docker` then `prebuild-devcontainer` workflows first
2. **Python venv not active**: Use `uv run` prefix for all Python commands
3. **Missing secrets**: Set `ZHIPU_API_KEY` in repo Settings → Secrets
4. **Template placeholders**: Replace `workflow-orchestration-queue-foxtrot54` and `intel-agency` in cloned instances

## State Machine (OS-APOW)

```
QUEUED → IN_PROGRESS → SUCCESS
                    ↘ ERROR
                    ↘ INFRA_FAILURE
                    ↘ STALLED_BUDGET
                    ↘ RECONCILING → IN_PROGRESS
```

| Label | Status | Terminal |
|-------|--------|----------|
| `agent:queued` | QUEUED | No |
| `agent:in-progress` | IN_PROGRESS | No |
| `agent:reconciling` | RECONCILING | No |
| `agent:success` | SUCCESS | Yes |
| `agent:error` | ERROR | Yes |
| `agent:infra-failure` | INFRA_FAILURE | Yes |
| `agent:stalled-budget` | STALLED_BUDGET | Yes |
