## Executive Summary

OS-APOW is a self-bootstrapping, event-driven AI orchestration system that transforms GitHub Issues into autonomous work orders. The system uses a 4-pillar architecture (Notifier, Queue, Sentinel, Worker) to manage AI agent workflows without human intervention.

## Key Architecture Decisions

### ADR-001: Python-Only Stack
No .NET/global.json. Pure Python 3.12+ ecosystem with uv package manager.

### ADR-002: Shell-Bridge Pattern
All container interactions via `devcontainer-opencode.sh` to ensure environment parity.

### ADR-003: Polling-First Resiliency
Polling is primary discovery; webhooks are optimization. Ensures self-healing on restart.

### ADR-004: Provider-Agnostic Queue
`ITaskQueue` interface enables future provider swapping (Linear, Jira, etc.).
